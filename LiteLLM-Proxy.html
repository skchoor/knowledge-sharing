Enroute : Claude Code ➜ LiteLLM Proxy ➜ NVIDIA NIM ( Free API Endpoints )

Overview:
LiteLLM serves as a translation gateway. It receives Anthropic-format requests from Claude Code (/v1/messages), translates them to OpenAI chat-completions format, and forwards them to NVIDIA’s NIM API.

This document provides a step‑by‑step guide for installing and configuring LiteLLM, setting up the environment, and enabling Claude Code. All commands are provided as‑is; copy them verbatim.

1. Prerequisites

  • Python 3.12+ (recommended)
  • uv package manager for isolated installations
  • MacOS / Guest OS of VirtualBuddy (tested on version 26.3)

2. Install LiteLLM

This installs LiteLLM with proxy support, which will be used as the backend for Claude Code.

uv tool install 'litellm[proxy]'

3. Configure Environment Variables ( $HOME/.zshrc )

Add the following lines to ~/.zshrc file and then reload the shell (source ~/.zshrc).

export LITELLM_USE_CHAT_COMPLETIONS_URL_FOR_ANTHROPIC_MESSAGES=true
export LITELLM_BASE_URL="http://127.0.0.1:4000"
export LITELLM_MASTER_KEY="freecc"

export NVIDIA_NIM_API_KEY="<nvapi-YOUR_KEY>"
export NVIDIA_NIM_BASE_URL="https://integrate.api.nvidia.com/v1"
export OPENROUTER_API_KEY="<sk-or-YOUR_KEY>"
export OPENROUTER_BASE_URL="https://openrouter.ai/api/v1"

export ANTHROPIC_BASE_URL="$LITELLM_BASE_URL"
export ANTHROPIC_AUTH_TOKEN="$LITELLM_MASTER_KEY"
export ANTHROPIC_API_KEY=""
export ANTHROPIC_DEFAULT_OPUS_MODEL="deepseek-v4-flash"
export ANTHROPIC_DEFAULT_SONNET_MODEL="qwen3.5-122b-a10b"
export ANTHROPIC_DEFAULT_HAIKU_MODEL="gemma-4-31b-it"
export CLAUDE_CODE_SUBAGENT_MODEL="qwen3.5-122b-a10b"

After editing, reload the shell:

source ~/.zshrc

4. Configure $HOME/litellm_config.yaml

model_list:
  - model_name: deepseek-v4-flash
    litellm_params:
      model: nvidia_nim/deepseek-ai/deepseek-v4-flash
      api_base: os.environ/NVIDIA_NIM_BASE_URL
      api_key: os.environ/NVIDIA_NIM_API_KEY
      rpm: 40 # Requests per minute
  - model_name: qwen3.5-122b-a10b
    litellm_params:
      model: nvidia_nim/qwen/qwen3.5-122b-a10b
      api_base: os.environ/NVIDIA_NIM_BASE_URL
      api_key: os.environ/NVIDIA_NIM_API_KEY
      rpm: 40
  - model_name: gemma-4-31b-it
    litellm_params:
      model: openrouter/google/gemma-4-31b-it:free
      api_base: os.environ/OPENROUTER_BASE_URL
      api_key: os.environ/OPENROUTER_API_KEY
      rpm: 20
  - model_name: default
    litellm_params:
      model: nvidia_nim/openai/gpt-oss-120b
      api_base: os.environ/NVIDIA_NIM_BASE_URL
      api_key: os.environ/NVIDIA_NIM_API_KEY
      rpm: 40
  - model_name: billable
    litellm_params:
      model: openrouter/deepseek/deepseek-v4-flash
      api_base: os.environ/OPENROUTER_BASE_URL
      api_key: os.environ/OPENROUTER_API_KEY
      rpm: 40

litellm_settings:
  drop_params: true
  use_chat_completions_url_for_anthropic_messages: true

general_settings:
  master_key: "freecc"

router_settings:
  fallbacks:
    - "deepseek-v4-flash": ["qwen3.5-122b-a10b", "gemma-4-31b-it", "default", "billable"]
    - "qwen3.5-122b-a10b": ["gemma-4-31b-it", "default", "billable"]
    - "gemma-4-31b-it": ["default", "billable"]
    - "default": ["billable"]

5. Create Startup Wrapper Script ( $HOME/Start_litellm.sh )

#!/bin/bash
source ~/.zshrc
export LITELLM_USE_CHAT_COMPLETIONS_URL_FOR_ANTHROPIC_MESSAGES=true

HOST="${${LITELLM_BASE_URL##*://}%:*}"
PORT="${LITELLM_BASE_URL##*:}"

exec /Users/vmm4/.local/bin/litellm \
  --config "$HOME"/litellm_config.yaml \
  --host "$HOST" --port "$PORT" \
  --model default "$@"

Make it executable:

chmod +x "$HOME"/start_litellm.sh

6. Auto‑Start on macOS (Automator Application)

  1. Open Automator → New Application.
  2. Add a Run Shell Script action with the following content:
  3. Select the Shell: choose /bin/bash
export PATH="$HOME/.local/bin:/opt/homebrew/bin:/usr/local/bin:/usr/bin:/bin"
export LITELLM_USE_CHAT_COMPLETIONS_URL_FOR_ANTHROPIC_MESSAGES=true
exec "$HOME"/start_litellm.sh >> "/var/log/litellm_$(date +%Y-%m-%d).log" 2>&1 &
  1. File → Save the Automator app as $HOME/Startup_LiteLLM.app and add it to System Settings → General → Login Items.
  1. Reboot the host and verify startup /var/log/litellm_(today).log

7. Verify the Setup

  1. OpenAI‑style test:
curl -X POST 'http://127.0.0.1:4000/chat/completions' \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $LITELLM_MASTER_KEY" \
  -d '{"model":"gpt-oss-120b","messages":[{"role":"user","content":"hello"}]}'
  1. LiteLLM models health check:
curl http://127.0.0.1:4000/health -H "Authorization: Bearer $LITELLM_MASTER_KEY"
  1. Anthropic‑style test (used by Claude Code):
curl -X POST http://127.0.0.1:4000/v1/messages \
  -H "Authorization: Bearer $LITELLM_MASTER_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model":"gpt-oss-120b","max_tokens":50,"messages":[{"role":"user","content":"hello"}]}'
  1. Bash commands to List, Stop and Restart automator's app ( Startup_LiteLLM.app ):
lsof -t -i :"${LITELLM_BASE_URL##*:}" -sTCP:LISTEN  			# LIST (PID)
kill -9 $(lsof -t -i :"${LITELLM_BASE_URL##*:}" -sTCP:LISTEN)  		# STOP (PID)
osascript -e 'quit app "Startup_LiteLLM"' && open -a Startup_LiteLLM	# RESTART

8. Install & Configure Claude Code

  1. Install the CLI globally:
npm install -g @anthropic-ai/claude-code
  1. Navigate to workspace/projects folder:
cd ~/obsivault/projects
  1. Start Claude Code with the desired model (e.g., default):
claude --model default
  1. Change the model inside Claude Code as needed:
/model <model name from LiteLLM gateway>
  1. Start a new session or clear the context:
/new OR /clear

9. Troubleshooting

  • 404 on /v1/messages – Ensure environment variable is set before LiteLLM starts.

    • *** LiteLLM’s /v1/messages endpoint (Anthropic format, which Claude Code uses) routes OpenAI-provider models through the Responses API by default. NVIDIA doesn’t have a Responses API, so requests return 404.     Setting export LITELLM_USE_CHAT_COMPLETIONS_URL_FOR_ANTHROPIC_MESSAGES=true forces LiteLLM to translate Anthropic messages → OpenAI chat completions format instead, which NVIDIA does support. ***

  • Authentication errors – Replace placeholder keys with real values in both ~/litellm_config.yaml and ~/.zshrc.
  • Login item runs without env vars – Always launch via the wrapper script created in step 5.

10. File Reference

File Purpose
~/.zshrc All env vars (NVIDIA key, LiteLLM gateway, model overrides)
litellm_config.yaml Model list, master_key, fallbacks
start_litellm.sh Wrapper that sets LITELLM_USE_CHAT_COMPLETIONS_URL_FOR_ANTHROPIC_MESSAGES then launches litellm
Startup_LiteLLM.app macOS Login Item (Automator app that runs start_litellm.sh on login)
